================================================================================
                    COMO USAR
================================================================================

INSTALACAO INICIAL (30 minutos)
  1. npm install -g appium
  2. appium-doctor
  3. Criar emulador/simulator
  4. pip install appium-python-client pytest

RODAR TESTES (5 minutos)
  1. appium &
  2. pytest test_mobile_example.py -v

GERAR RELATORIO HTML
  pytest test_mobile_example.py -v --html=report.html

RODAR TESTE ESPECIFICO
  pytest test_mobile_example.py::TestLoginAndroid::test_login_success -v

                    COMANDOS RAPIDOS
================================================================================

Iniciar Appium Server:
  appium --address 127.0.0.1 --port 4723

Rodar testes (Console):
  pytest test_mobile_example.py -v

Rodar com relatorio HTML:
  pytest test_mobile_example.py -v --html=report.html

Rodar em paralelo:
  pytest test_mobile_example.py -n auto

Rodar apenas smoke tests:
  pytest test_mobile_example.py -m smoke

Rodar com retry:
  pytest test_mobile_example.py --reruns 3

Verbose com logs:
  pytest test_mobile_example.py -vv -s

================================================================================
                    DICAS IMPORTANTES
================================================================================

1. SEMPRE USE WAITS
   RUIM: time.sleep(2)
   BOM: WebDriverWait(driver, 10).until(...)

2. PAGE OBJECT PATTERN
   RUIM: find_element diretamente em testes
   BOM: Centralizar locators em page objects

3. DATA SEPARATION
   RUIM: Dados hardcoded nos testes
   BOM: Usar config.json ou fixtures

4. UNIQUE LOCATORS
   RUIM: //*[text()='Login']
   BOM: //button[@resource-id='com.app:id/login']

5. SCREENSHOTS ON FAILURE
   OK: Capturar automático em caso de erro

6. CI/CD INTEGRATION
   OK: Rodar testes em cada PR

7. CLOUD SERVICES
   OK: Testar em múltiplos devices/versões
